# 데스크탑 폴더 삭제 전 체크리스트 (myungni_next)

## 1) 삭제 전 필수 확인
1. 현재 작업 폴더: `C:\Users\chris\Desktop\myungni_next`
2. Git 상태 확인: `git status`
3. 원격 확인: `git remote -v`
4. 최신 반영 여부: `git log --oneline -n 5`
5. 필요 시 백업 ZIP 1개 생성

## 2) 안전 삭제 조건
1. `git status` 결과가 깨끗함(또는 필요한 변경 모두 커밋/푸시 완료)
2. GitHub 저장소에서 최신 커밋 확인 완료
3. Vercel 배포 URL 접속 정상 확인
4. 로컬 전용 산출물(이미지, 엑셀, PDF) 백업 완료

## 3) 삭제하면 사라지는 것
1. 데스크탑 로컬 파일 전체
2. 커밋하지 않은 수정본
3. 로컬 테스트 산출물/임시 파일

## 4) 삭제해도 유지되는 것
1. GitHub에 이미 푸시된 커밋
2. Vercel에 이미 배포된 서비스
3. 원격 저장소의 브랜치/태그 이력

## 5) 삭제 후 복구 방법
1. 원하는 위치에 새 폴더 생성 (예: `C:\dev`)
2. `git clone <저장소URL>`
3. `npm install`
4. `npm run dev` 또는 배포 점검 스크립트 실행

## 6) 최종 판단
- 위 2) 항목이 모두 충족되면 삭제 가능
- 하나라도 미충족이면 삭제 보류
